# T0 dart2wasm mechanism audit

The structural oracle is `dart-lang/sdk` at
`898a1e4bbfbc472dc0a9505dc7d2e4c21d6f856e`. References below are file and
symbol names at that exact revision. This audit compares compiler mechanisms;
it does not treat Dart behavior as an oracle for Julia semantics.

| Snapshot delivery | Outside dart2wasm's compiler scope. | **Integration-only evidence.** Snapshot is downstream evidence that certified prerequisites survive notebook extraction and real browser execution. | Snapshot cannot waive a compiler diagnostic or turn an unsupported method into a compiler claim. | Split HTTP and single-file exports must be full islands in both engines; the deliberately unsupported dynamic call must produce the exact declared static-fallback diagnostic. |

T0 therefore certifies only MOI value/runtime prerequisites. It does not certify
JuMP model construction, MOI model storage, bridge dispatch, a solver, reified
generics, FFI, threads, async behavior, finalizers, or native libraries. Each
later milestone must add a new pinned dart2wasm audit row for every mechanism it
begins to use.
