## gap `c59895cfb734` ‚Äî composition corrupts `maximum(sort(...))` (scratch-local aliasing)

**Category:** `wrong_value` ‚Ä¢ **Surface:** vector ops (Int64)

**Minimal reproducer:** `repro(x) = sum([0,0,0]) + maximum(sort([0, x, x]))`
at `x = -1`: native `0`, wasm `-1`.

**This is a composition-only bug ‚Äî every individual op is correct.** Narrowing
(`/tmp` probes, all over the standard sample inputs):

| expression | result |
|---|---|
| `maximum(sort([0,x,x]))` alone | ‚úÖ ok |
| `maximum([0,x,x])` (no sort) | ‚úÖ ok |
| `0 + maximum(sort([0,x,x]))` | ‚úÖ ok |
| `sum([0,0,0]) + maximum([0,x,x])` (no sort on right) | ‚úÖ ok |
| `sum([0,0,0]) + maximum(reverse([0,x,x]))` | ‚úÖ ok |
| `sum([0,0,0]) + sum(sort([0,x,x]))` | ‚úÖ ok |
| `sum([0,0,0]) + first(sort([0,x,x]))` | ‚úÖ ok |
| `maximum(sort([0,x,x])) + sum([0,0,0])` (swapped) | ‚úÖ ok |
| `sum([0,0,0]) * maximum(sort([0,x,x]))` | ‚úÖ ok |
| **`sum([0,0,0]) + maximum(sort([0,x,x]))`** | ‚ùå wrong |
| **`prod([9,9,9]) + maximum(sort([0,x,x]))`** | ‚ùå wrong (729 vs 728) |

**Hypothesis:** a fixed/shared **scratch local** is reused without save-restore
across two vector operations evaluated in sequence. The trigger is precisely:

> a **left operand** that is a vector reducer over a literal (`sum`/`prod`),
> **then** a **right operand** of the form `maximum(sort(...))`.

`maximum(sort(...))` returns the array's *minimum* (the first element of the
ascending sort) under these conditions ‚Äî i.e. `maximum` reads the wrong slot/local
after the left operand has written the shared scratch. Swapping operand order, or
removing either the left vector op or the right `sort`, avoids the clobber.

**Where to look:** `CompilationContext.scratch_locals` (allocated once, `context.jl`),
and the `sort` insertion-sort overlay + `maximum` reducer codegen ‚Äî check whether
they claim overlapping scratch local indices and whether intermediate results are
spilled across a sub-expression that also touches those locals. The `+` vs `*`
difference and the order-sensitivity both point at operand-evaluation interleaving
with shared mutable scratch.

**Verify the fix:** `julia --project=test/fuzz test/fuzz/run.jl verify` (auto-closes
the gap when the reproducer runs clean), then re-fuzz at higher depth.
