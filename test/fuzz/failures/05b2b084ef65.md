## Analysis
**ROOT-CAUSED → TIER-1 (dynamic dispatch), 2026-06-17 loop.** Minimal trigger:
`Dict(Int32(0) => 0, 0 => 0)` → **`Dict{Signed, Int64}`** (`Signed = typejoin(Int32,
Int64)`, ABSTRACT). The `sum`/`abs`/`cumsum` wrappers the shrinker keeps producing are
just disguises for "a second key of a different Int width", which widens the inferred key
type to the abstract `Signed`. Native returns `1` (both keys are `0`, equal); wasm traps.

This is the shared root of the whole ~11-gap "abstract-Dict-key (Int-widen)" cluster
(`run.jl rank` #1): `01e4e1bb61d3 05b2b084ef65 20775d3ec5b2 418e48316c5c 46a251368b26
a70595550bd8 b85535ae7df7 cb567b610585 e6c8c6edaeaa edf2c277a170 ef3c54645d9f`.

Why it's NOT a clean Tier-0 fix: an abstract-`Signed` key is boxed, and Base `Dict`'s
`ht_keyindex`/insert call `hash(key)` and `isequal(k1,k2)` on it — which must DISPATCH to
the concrete type (`hash(::Int32)` vs `hash(::Int64)`). WT can't resolve that dynamic
call → trap. So closing this cluster needs **Tier-1 dynamic dispatch** (the gated/perturbing
`WT_DYNDISPATCH` frontier), OR a narrow special-case (abstract-but-all-integer key →
represent boxed key, unbox to i64 for a consistent hash/eq) — intricate, deferred.
Re-tiered from T0→T1; folded into the T1 dispatch frontier (task #33). See LOOP.md §1-2.

**UPDATE 2026-06-21 — the overlay approach does NOT work; analysis above is incomplete.**
Tried `@overlay Base.hash(::Signed,::UInt)` + `isequal(::Signed,::Signed)` unboxing the key to
Int64 via an isa-ladder (the isa-ladder itself is verified to compile+run correctly in WT —
`x isa Int32 ? Int64(x) : …` works). It had ZERO effect: the helper never appears in the
compiled module, and BOTH the constant-keyed (`Dict(Int32(0)=>0,0=>0)`) AND runtime-keyed
(`repro2(a::Int32,b::Int64)=length(Dict(a=>0,b=>0))`) cases still trap. Crucially:
**`WT.compile(...; strict=true)` COMPILES CLEAN** for both — so this is NOT a compile-time
"can't resolve hash(::Signed)" dispatch failure (which strict would raise). It's a **runtime
`unreachable`** reached inside WT's *compiled* Dict construction/indexing logic for abstract
`Signed`-keyed storage (hashindex / ht_keyindex2_shorthash! / Memory{Signed} slot handling).
i.e. WT compiles the abstract-keyed Dict but the emitted indexing logic is wrong → traps. The
real fix needs debugging the compiled Dict internals (find which of the 10 `unreachable`s in
the ~5KB module is hit — the bridge gives no JS stack trace, so needs instrumentation), NOT a
hash/isequal overlay. Deferred: deeper than first thought, low PI relevance (mixed-int-width
keys are a fuzzer artifact). Repros: /tmp/dict_repro2.jl, /tmp/dict_runtime.jl, /tmp/isa_probe.jl.

**UPDATE 2026-06-22 (wt-soundness-loop-4) — INSTRUMENTED, root pinned. It is NOT "wrong
indexing logic" — WT stubs the WHOLE Dict CONSTRUCTION.** Node trap-stack
(`Error.stackTraceLimit=Infinity`) on the minimal `length(Dict(Int32(0)=>0, 0=>0))` →
`wasm-function[1]:0x287`; `wasm-tools print --print-offsets` shows the entire `$repro` body is
`global.get 56; unreachable; unreachable; unreachable; unreachable`. The trim IR is:
```
%1 = dynamic invoke Base._compute_eltype(Tuple{Pair{Int32,Int64},Pair{Int64,Int64}})::Type  # → Pair{Signed,Int64}
%2 = dynamic Base.dict_with_eltype(#Dict##0#, (0=>0, 0=>0), %1)::Dict   # build Dict{Signed,Int64}
%3 = getfield(%2, :count); return %3
```
WT cannot construct an ABSTRACT-keyed `Dict{Signed,Int64}` → it stubs `dict_with_eltype` to
`unreachable`. (`_compute_eltype` is type-level but the cor-style concrete-eval fix does NOT fold
it — it's a `dynamic invoke` and stays `::Type`, not `Const`; even if it folded, the
`dict_with_eltype` call takes the runtime tuple of pairs so it's not all-Type and won't fold.)
REAL FIX (deep, multi-cycle, low-PI): abstract-keyed-Dict storage codegen — either the Tier-1
dynamic-dispatch frontier (`hash(::Signed)`/`isequal` resolve dynamically), OR a narrow
special-case: when an abstract key type's concrete instances are ALL `<:Integer`, store keys
unboxed as i64 (width-extend Int32→Int64) so hash/eq are concrete i64 ops (value-/equality-
preserving for the length/isempty/haskey/get ops these gaps use — but changes the Dict TYPE, so
soundness must be checked). Instrument: /tmp/dict_instrument.jl + /tmp/dict_run.cjs.
