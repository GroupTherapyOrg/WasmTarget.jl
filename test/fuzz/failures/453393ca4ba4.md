Minimal repro (FuzzHarness):

```julia
function entry7(n::Int64)
    straight(x) = x + 1.0
    @noinline function sn(f, m)   # captures straight → closure; @noinline forces the cross-call
        straight(f(Float64(m)))
    end
    f(x) = x * 2.0
    sn(f, n)
end
compile_and_run(entry7, (Int64,), [(5,)])  # → (:trap, "unreachable"); native 11.0
```

Mechanism (proven in the PlutoIslands pipeline with WT_LOG_REGISTRY=1): the callee is
registered as `(typeof(sn){typeof(straight)}, argtypes...)` — self-type prepended — and
compiled with the closure box as wasm param 1. The call-site cross-call resolution finds
the entry via the PURE-320 self-prepended retry but emits the call WITHOUT pushing the
closure object: 6 values for a 7-param functype. Depending on downstream layout this
either fails validation ("expected (ref null $type) but nothing on stack" — PlutoIslands
newton at 0x1fd2) or validates-then-traps (this repro).

Fix direction: when the resolved registry entry's arg_types carry the prepended self,
compile and push the function-position value (the closure instance — an SSA/local the
call site already has) before the explicit args; alternatively register closure callees
under their captured-instance identity and emit the env push uniformly.
