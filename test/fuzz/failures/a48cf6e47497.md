## Analysis
**Root cause: shared scratch-local aliasing across operand evaluation** (a
local-allocator bug, not an op bug). Narrowing ‚Äî every sub-piece is correct in
isolation:

| expression | result |
|---|---|
| `lcm(2,x)` | ‚úÖ |
| `cumsum([0,x,x])`, `unique(cumsum([0,x,x]))`, `minimum(unique(cumsum([0,x,x])))` | ‚úÖ |
| `minimum(cumsum([0,x,x])) \| lcm(2,x)` (no `unique`) | ‚úÖ |
| `minimum(unique([0,x,x])) \| lcm(2,x)` (no `cumsum`) | ‚úÖ |
| `minimum(unique(reverse([0,x,x]))) \| lcm(2,x)` (`reverse`, not `cumsum`) | ‚úÖ |
| `lcm(2,x) \| minimum(unique(cumsum([0,x,x])))` (**operands swapped**) | ‚úÖ |
| **`minimum(unique(cumsum([0,x,x]))) op lcm(2,x)`** (op = `\|` *or* `+`) | ‚ùå |

So the trigger is precisely: a **left** operand of the shape `unique‚àòcumsum`
(two chained vector allocations) **then** a **right** operand `lcm`. It is
**order-sensitive** (swapping fixes it) and **operator-agnostic** (`|` and `+`
both fail) ‚Üí classic operand-evaluation interleaving with a shared mutable local.

**Traced in the WAT** (`f(x)=minimum(unique(cumsum([0,x,x]))) + lcm(2,x)`, one
1668-line monolithic function): the final `i64.add` reads `local 59` (= A,
correct) and `local 66` (= B = `lcm`). For x=1, native = `0 + 2`; wasm = `0 + 0`,
so **B is wrong**. lcm = `2 * (x √∑ gcd(2,x))`; the param `x` (local 0) is never
written, so the bad factor is **gcd(2,1) returning 2 instead of 1**. gcd is the
binary-GCD `oddpart << commonpow2`; one of its state locals carries a stale value
left behind by the preceding `unique‚àòcumsum` code ‚Äî i.e. the gcd loop and the
cumsum/unique loops were assigned **overlapping local indices** without a spill
across the operand boundary.

**Where to fix:** the local allocator / `CompilationContext` local reuse ‚Äî ensure
a sub-expression's working locals don't alias another sub-expression's live locals
across operand evaluation. This is the same *class* as the now-fixed
`maximum(sort(...))` cluster (which my explicit-loop `maximum`/`minimum` overlays
sidestepped); the proper allocator fix would close this remaining instance for
good. **Deferred to Part 2** (deep codegen, not overlay-fixable).
