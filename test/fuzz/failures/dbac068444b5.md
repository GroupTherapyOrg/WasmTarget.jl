## Analysis
**Root cause: duplicate-key `Dict` construction with String keys returns garbage.**
Narrowed — single-key String-Dict `get` all work:
`get(Dict("a"=>5),"a",0)`=5 ✅, `get(Dict(""=>7),"",0)`=7 ✅,
`get(Dict(""=>0),"x",0)`=0 ✅ (missing). The failure is specifically the **duplicate
empty-string keys** `Dict("" => 0, "" => 0)`: `get(…, "", 0)` should return `0`, but
wasm returns a **negative** value (so `sign(0+…)` = -1 vs native 0).

A Dict literal with duplicate keys should collapse to one entry (last wins); the wasm
Dict-construction path mishandles the duplicate insert (String-key hash/slot reuse),
leaving a slot whose read yields a stale index/garbage rather than the value. Same
family as the String-keyed Dict codegen and [[627592b54cf2]] (also duplicate keys).
**Where to fix:** duplicate-key handling in Dict construction codegen. **Deferred to
Part 2** (deep Dict codegen).
