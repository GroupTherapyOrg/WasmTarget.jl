## Analysis

**FIXED (P3 bugsmash).** Cousin of ff6dc9760825 with the inner div
depending on `x`: the catch arm then captures `x` across the exception
edge (1.13 emits ϒ/φᶜ for it) and — the load-bearing difference — the
chain has TWO merge phis: φ13 internal to level 2 (`φ(%11 => %9, %12 =>
0x00)`) and φ15 the outer merge (`φ(%6 => %4, %14 => %13)`).

The first (flat) generate_catch_try_chain_merge assumed ONE shared
merge phi: it stored only φ13's edges, so level 1's normal exit (%4 via
edge 6 → φ15) was never stored and φ15's local read its zero default —
silent wrong value 0 for every non-throwing input.

Fix: made the generator recursive (`_emit_merge_chain_level!`). Each
level finds its OWN merge phi (first phi past its catch dest with a
try-side edge from its body), wraps body+arm in its own $merge block,
and the next level nests inside the catch arm. Return-style levels
(no merge phi) emit the plain block and recurse. Second bug caught
while testing: the leaf arm must NOT get the stackifier's trailing
unreachable pad — it falls through into the enclosing level's
catch-side phi stores (the pad trapped ff6d's whole catch path on
1.13). Full matrix (both shapes × {0,1,5,255} × both versions) green.
