## Analysis
**Root: `sinh`/`cosh`/`tanh` value-stub (hyperbolic feature gap).** Not a hypot
bug — the `hypot(Float64,Float64)` overlay already returns `Inf` correctly when
either operand is infinite. The validation error "expected f64 but nothing on
stack" is the **second** operand `sinh(x)` producing no value: `sinh`/`cosh`/`tanh`
have no overlay and aren't natively codegen-able, so they emit a value-stub
(nothing on the stack) → the surrounding `hypot` call then has a missing f64
operand → module fails to validate.

**FIXED** — added Float64 overlays (`interpreter.jl`). `cosh(x) = (eᵃ+e⁻ᵃ)/2`
exact; `sinh` uses a Taylor branch for |x| < 0.35 (else `eˣ-e⁻ˣ` cancellation
blows past `rtol=1e-9` in the band `[1e-12,1e-7]`; below `1e-12` the `atol`
covers it); both use an overflow-safe `eᵃ/2 = exp(a-ln2)` for |x| > 20 so e.g.
`sinh(710)` stays finite (1.117e308) instead of overflowing to Inf; `tanh =
sinh/cosh` with `|x|>20 ⇒ ±1`. Float32 redirects through Float64. Verified 42/42
vs native across the sweep; guarded by `@testset "Hyperbolic sinh/cosh/tanh"`.
Still open (same value-stub class, next batch): `expm1/log1p/sinpi/cospi/
deg2rad/rad2deg`.
