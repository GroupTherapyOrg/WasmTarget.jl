## Analysis

**FIXED (P3 bugsmash).** Inference folds the constant outer/inner catch
arms, leaving a 2-level catch-try chain whose normal paths `goto` a
merge phi (`φ(%5 => %3, %10 => 0x00)`) instead of returning:
Enter(2,cd=6) / body div(x,x) / leave+goto 11; Enter(6,cd=9) / body
div(0,0)+unreachable; catch tail pop_exceptions; phi; return.

Dispatch hole: not "nested" (inner.enter == outer.catch_dest), and the
return-style chain generator rejects it via the cross-phi guard (edge
%5 < last catch_dest). It fell to `generate_sequential_try_catch`,
which emitted level 2 AFTER level 1's catch landing block instead of
inside the handler — the handler was empty, so the result block closed
with nothing on the stack (`expected i32 but nothing on stack`).

Fix: new `generate_catch_try_chain_merge` — same per-level layout as
the return-style chain, but each level's normal completion stores its
try-side merge-phi edge values and `br`s to a shared $merge block
(the N-level generalization of the single-region has_merge structure);
the final catch arm falls through into the merge with its own phi
stores. Dispatch guards: merge phi exists with a try-side edge, no
branch over the outer region, no inter-level over-branches, and every
try-side edge originates inside some level's body.
