## Triage — KNOWN LIMITATION (P3 bugsmash, 2026-06-12)

Heterogeneous-key Dict literals (`Dict(Int32(0) => 0, sum(Int32[...]) => 0)`
— the computed key is Int64, so the pairs are Pair{Int32,Int64} +
Pair{Int64,Int64}) route through Base.dict_with_eltype/_compute_eltype,
whose inferred result is the UNPARAMETERIZED `Dict` — every downstream
operation then needs runtime type dispatch, which WasmGC codegen does not
support (same root as heterogeneous-union vararg tuple reads, see
284d3e7059cd triage). Fixing requires either a dynamic-dispatch runtime or
inference-level monomorphization of the promoted Dict type. Documented in
README Known Limitations; left open-triaged intentionally.
**Workaround:** promote keys explicitly so all pairs share one type:
`Dict(Int64(k1) => v1, k2_int64 => v2)`.
