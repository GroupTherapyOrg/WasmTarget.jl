## Triage (P3, 2026-06-12)

Reproduces standalone (`ca(x::UInt8) = try div(0x00, x-x) catch; isempty([Int32(0),Int32(0),Int32(0)]) ? (try div(0x00,0x00) catch; 0x00 end) : x end`,
x=7 → native 7, wasm 0). IR shape: outer region (enter#2→catch#5); the
CATCH ARM contains `goto #16 if not %31` (the isempty branch) followed by
the INNER region (enter#13→catch#15) and the else tail #16. The GotoIfNot
branches OVER the inner region (dest #16 > inner.catch_dest #15) — i.e. a
branch-split WITHIN a catch arm. The chain dispatch (`_chain_consecutive`)
still selects the catch-containing-try CHAIN layout, whose level-2 prefix
(blocks #5..#12) ends in the out-of-subset branch: the exit-block path and
the fall-through path both continue into the inner try_table, so the
else arm (`return x`) is unreachable and wasm returns the inner-catch 0.

Fix direction: chain dispatch must reject prefixes that branch over the
inner region (a `_branches_over_inner` analogue of P2-batch22's
`_branches_over_outer`), and the shape needs branch-split-in-catch-arm
handling (generate_branch_split_try nested inside the catch arm, or
recursive try-region dispatch when emitting catch arms).
Likely same family: 464d3b1b41ec, 0d6aa24022dd, ae64a1ba676e.
