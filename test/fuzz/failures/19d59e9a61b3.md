## Analysis
**Root cause: `string(::Float64)` is unimplemented (needs Ryu).** The `Set`/`isempty`
machinery is fine — `isempty(Set(["",""]))` ✅. The trap is the `string(0.0)`
element: confirmed `ncodeunits(string(0.0))`, `ncodeunits(string(1.5))`, and
`string(float(x))` all trap, while every non-float piece works.

Float→string is the shortest-round-trip formatting algorithm (`Base.Ryu`), a real
**feature to implement**, not an overlay sidestep: it needs the big-integer /
table-driven digit generation Ryu uses. (Contrast `string(::Int64)`, which is
fixed and compiles via the real Base path — digit extraction.)

**Where to fix:** implement a WasmGC-friendly `Base.Ryu.writeshortest` (or a
correct fixed-format fallback) for Float32/Float64, then wire `string(::AbstractFloat)`
/ `print(::AbstractFloat)` to it. **Deferred to Part 2** (feature work). Until then
the fuzzer's float→string surface is a known limitation; see `FINDINGS.md`.
