## Analysis
**Root cause: non-ASCII `Char` category lookup needs Unicode tables (feature).**
`isspace('é')` is `false` natively (é isn't whitespace) but **traps** in wasm: the
`Char` whitespace test for codepoints ≥ 0x80 routes through a Unicode category path
that's unimplemented. ASCII works (`isspace(' ')` → true, verified ✅).

Same root as [[ff2542499cc2]] (`uppercase('é')`) and [[6bd3c602cf17]]
(`isdigit('é')`): all three need Unicode character-property tables. **Deferred to
Part 2** (feature — Unicode tables, not overlay-fixable).
