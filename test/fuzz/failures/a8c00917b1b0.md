Minimal repro (FuzzHarness):

```julia
function range_repro(n::Int64, x0::Int64)
    r = -10:0.01:10
    Float64(first(r)) + Float64(last(r)) + Float64(step(r)) + Float64(n) + Float64(x0)
end
compile_and_run(range_repro, (Int64, Int64), [(6, 5)])  # → (:trap, "unreachable"); native 11.01
```

Disassembly of the emitted export: three f64.const (-10, 0.01, 10) then a wall of
`unreachable` — codegen abandons the twiceprecision/colon path and pads with traps
instead of refusing in strict mode. WASM_FINDINGS #7 (StepRangeLen iteration) is the
same family; this is the constructor+first/last/step flavor. Blocks newton.jl's two
canvas groups (and any notebook passing a float range as an argument).

Expected fix: either compile Base.range/TwicePrecision honestly (preferred — ranges
are everywhere in notebooks) or refuse at compile time with a named unsupported-op
error so the pipeline degrades with the real reason instead of trapping at runtime.
