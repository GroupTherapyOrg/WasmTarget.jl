## Triage (P3 bugsmash, 2026-06-12)

Plain-element matrix literals ([1.0 2.0; 3.0 4.0]) work after the
matrix-layout fix (gap 3aaa51b9a688). The TUPLE-element shape routes
through Base._typed_hvncat_dims/_typed_hvncat_shape (1049-stmt
machinery, concrete rettype) which currently stubs → runtime trap.
Whitelisting those helpers was attempted and REVERTED: compiling them
sends the discovery/compile pipeline into a StackOverflow (recursive
type registration or discovery loop on the NTuple-element matrix
types) — a compiler crash, strictly worse than the trap. Needs the
recursion diagnosed (stacktrace via ulimit/smaller depth) before the
whitelist can land. Workaround for consumers: build the matrix with
Matrix{NTuple{4,Float64}}(undef, m, n) + explicit stores (works).
